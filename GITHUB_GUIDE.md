## الجزء الثاني: تشغيل قاعدة بيانات Supabase وتوصيلها 🗄️

المنصة الآن مجهزة بالكامل لتدعم المزامنة السحابية الفورية مع **Supabase** مع الحفاظ على العمل أوفلاين في حالة غياب الإعدادات. لتفعيل Supabase:

### 1. إنشاء مشروع جديد في Supabase
1. اذهب إلى موقع [Supabase](https://supabase.com) وسجل دخولك أو أنشئ حساباً مجانياً.
2. اضغط على **New Project** واختر منظمتك.
3. اكتب اسماً للمشروع (مثال: `a-class-db`).
4. اكتب كلمة مرور قوية لقاعدة البيانات (Database Password) واحفظها جيداً.
5. اختر المنطقة الجغرافية الأقرب لجمهورك (مثال: `EU (Frankfurt)` أو `Middle East`).
6. اضغط على **Create new project** وانتظر دقيقتين حتى يكتمل إنشاء المشروع.

### 2. تشغيل جداول المنصة والبيانات التأسيسية (SQL Script)
1. في لوحة تحكم مشروعك بـ Supabase، توجه إلى القائمة الجانبية واضغط على **SQL Editor**.
2. اضغط على **New query** (مستند فارغ جديد).
3. افتح ملف `supabase_schema.sql` الموجود في جذر هذا المشروع، وانسخ كل محتواه بالكامل.
4. الصق المحتوى داخل الـ SQL Editor في Supabase.
5. اضغط على زر **Run** باللون الأخضر في الزاوية العلوية اليمنى.
6. ستظهر لك رسالة تفيد بنجاح التنفيذ وتأسيس الجداول والعلاقات وإضافة المدرسين والمواد الافتراضية بنجاح! 🎉

### 3. ربط المنصة بمشروعك (Environment Variables)
1. في Supabase، اذهب إلى الإعدادات **Project Settings** ثم اضغط على **API**.
2. ستجد حقلين رئيسيين:
   * **Project URL**: رابط مشروعك الخاص.
   * **anon (public) API Key**: مفتاح الـ API العام.
3. قم بإنشاء ملف `.env` في المجلد الرئيسي لمشروعك (على جهازك المحلي)، والصق القيم كالتالي:

```env
VITE_SUPABASE_URL=رابط_المشروع_الخاص_بك_هنا
VITE_SUPABASE_ANON_KEY=مفتاح_api_العام_الخاص_بك_هنا
```

4. أعد تشغيل السيرفر المحلي (`npm run dev`)، وستجد أن المنصة الآن تقوم بالقراءة والكتابة والحفظ مباشرة في قاعدة بيانات **Supabase** الحية!

## مميزات هذا الدمج الذكي 🌟
* **مزامنة البيانات الفورية**: عند إضافة معلم أو مادة أو تفعيل كود شحن، يتم إرسالها وحفظها فورياً في السحاب.
* **المرونة والتجربة السلسة**: إذا لم يتم العثور على ملف `.env` أو لم يتم ملء البيانات، تعمل المنصة بشكل تلقائي على الذاكرة المحلية للمتصفح دون أي أعطال أو شاشات بيضاء.
